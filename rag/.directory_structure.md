# RAG 目錄結構說明

## 📂 目錄組織

整理後的目錄結構清晰明了，分為以下幾個主要部分：

### 🎯 核心代碼（根目錄）

主要的 Python 模塊，是系統的核心：

- `rag_bot.py` - **主程序**，互動式聊天機器人 CLI
- `rag_engine.py` - RAG 核心引擎
- `vector_store.py` - 向量存儲和檢索
- `document_processor.py` - 文檔處理和分塊
- `config.py` - 配置管理
- `__init__.py` - Python 包初始化

### 📂 data/

存放數據文件：

- `demo_docs/` - 演示用的文檔（.txt 文件）
  - python_basics.txt
  - machine_learning.txt
  - rag_systems.txt
  - wovenid.txt

### 📚 docs/

所有文檔資料：

- `QUICKSTART.md` - 快速開始指南
- `DEEP_DIVE.md` - 深入技術解析（第 1 部分）
- `DEEP_DIVE_PART2.md` - 深入技術解析（第 2 部分）
- `DEEP_DIVE_PART3.md` - 深入技術解析（第 3 部分）
- `chunk_logic_explained.md` - 分塊邏輯詳細說明
- `index_flow_diagram.md` - 索引流程圖（Markdown）
- `index_flow_diagram.html` - 索引流程圖（互動式 HTML）

### 💡 examples/

可執行的示例代碼：

- `example_usage.py` - 展示如何程序化使用 RAG 系統

### 🎨 demos/

演示腳本：

- `chunk_demo.py` - 分塊邏輯的詳細演示
- `chunk_simple_demo.py` - 分塊邏輯的簡化演示

### 🔧 scripts/

工具腳本：

- `setup.sh` - 環境設置腳本
- `demo.sh` - 快速演示腳本
- `verify_k8s_models.sh` - Kubernetes 環境模型驗證
- `test_k8s_connection.py` - Kubernetes 連接測試

### 🔒 系統文件

- `venv/` - Python 虛擬環境（不納入版本控制）
- `__pycache__/` - Python 緩存（自動生成）

## 🎯 快速導航

### 想要...

- **開始使用** → 運行 `python rag_bot.py`
- **了解原理** → 閱讀 `docs/DEEP_DIVE.md`
- **查看示例** → 查看 `examples/example_usage.py`
- **理解分塊** → 閱讀 `docs/chunk_logic_explained.md` 或運行 `python demos/chunk_simple_demo.py`
- **查看流程** → 在瀏覽器中打開 `docs/index_flow_diagram.html`

## 📏 文件大小參考

- 核心代碼: ~30KB（4 個主要 Python 文件）
- 文檔: ~80KB（7 個 Markdown/HTML 文件）
- 演示數據: ~3KB（4 個 .txt 文件）
- 腳本工具: ~10KB（4 個腳本）

## ✅ 整理原則

1. **核心代碼在根目錄** - 便於 import
2. **文檔集中存放** - docs/ 目錄
3. **示例分類** - examples/ 用於代碼，demos/ 用於演示
4. **腳本獨立** - scripts/ 目錄存放工具腳本
5. **數據分離** - data/ 目錄存放數據文件

## 🎓 學習路徑建議

### 新手

1. 閱讀 `README.md`
2. 閱讀 `docs/QUICKSTART.md`
3. 運行 `python rag_bot.py` 試用
4. 運行 `python demos/chunk_simple_demo.py` 理解分塊

### 進階

1. 閱讀 `docs/DEEP_DIVE.md`（所有 3 部分）
2. 查看 `docs/index_flow_diagram.html`
3. 研究核心代碼
4. 修改參數試驗

### 開發者

1. 研究所有核心模塊的源碼
2. 運行 `examples/example_usage.py`
3. 修改並擴展功能
4. 添加測試和文檔
